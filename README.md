## 📌 О проекте

Уютное мобильное приложение для любителей готовить, которое объединяет поиск рецептов, ведение кулинарного дневника и планирование питания с автоматическими списками покупок.

---

## 🚀 Функционал

* Поиск рецептов по названию и ингредиентам
* Фильтрация рецептов по категориям блюд и национальным кухням
* Сохранение рецептов с присвоением статуса («Хочу приготовить» / «Уже приготовил»)
* Создание собственных кулинарных рецептов с нуля
* Планирование персонального меню (питания) на неделю
* Автоматическое формирование списка покупок на основе выбранных блюд

## 🧠 Архитектура

### 📱 iOS

Используется архитектура **MVVM (Model-View-ViewModel)**

* Model — данные
* View — интерфейс
* ViewModel — логика

Дополнительно:

* Service Layer (NetworkService, FirebaseService)

---

### 🤖 Android

Используется архитектура **MVVM**

Слои:

* **Data** — работа с данными (API, Room, Firebase)
* **Domain** — бизнес-логика (Use Cases)
* **UI** — экраны на Jetpack Compose с ViewModel

Дополнительно:

* Repository Pattern — единый источник данных
* DI через Hilt — внедрение зависимостей
* Single Activity — всё приложение работает в одной Activity

## 🔄 Основные API-запросы

1. **Поиск по названию**  
   `GET /search.php?s={название_блюда}`  
   Используется, когда пользователь вводит название в поисковую строку. Возвращает все рецепты, в названии которых встречается запрос.

2. **Фильтр по категории**  
   `GET /filter.php?c={категория}`  
   Возвращает все рецепты из указанной категории (Dessert, Breakfast, Seafood и т.д.)

3. **Фильтр по национальной кухне**  
   `GET /filter.php?a={кухня}`  
   Возвращает все рецепты конкретной кухни (Italian, Japanese, Russian и т.д.)

4. **Фильтр по ингредиенту**  
   `GET /filter.php?i={ингредиент}`  
   Возвращает все рецепты, в которых используется указанный ингредиент.

5. **Детальная информация о рецепте**  
   `GET /lookup.php?i={id_рецепта}`  
   Возвращает полные данные по конкретному рецепту: ингредиенты с мерами, инструкцию, видео.

6. **Случайный рецепт**  
   `GET /random.php`  
   Возвращает один случайный рецепт.

7. **Список всех категорий**  
   `GET /categories.php`  
   Возвращает все доступные категории с картинками и описаниями.

8. **Список всех национальных кухонь**  
   `GET /list.php?a=list`  
   Возвращает список всех доступных кухонь.

9. **Список всех ингредиентов**  
   `GET /list.php?i=list`  
   Возвращает список всех ингредиентов (для автодополнения).

* Services

## ▶️ Запуск проекта

### iOS

1.  Открыть `CookingAssistant.xcodeproj` в Xcode (версия 15.0 или новее).
2.  Убедиться, что зависимости через SPM (Swift Package Manager) подтянулись автоматически.
3.  Выбрать целевое устройство (симулятор или реальный iPhone).
4.  Нажать `Cmd + R` для запуска.

### Android

1. Открыть проект в **Android Studio**.
2. Дождаться синхронизации Gradle и подтягивания зависимостей.
3. Убедиться, что в файле `local.properties` указан путь к SDK.
4. Выбрать целевое устройство (эмулятор с API 29+ или реальный телефон).
5. Нажать `Shift + F10` для запуска.
